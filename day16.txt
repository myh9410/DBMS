# 모델링 프로세스 : 모델링을 할 때 어떤 단계를 통해 모델을 만들어내는지를 의미함.

# 모델링 프로세스 모델 중에 폭포수 모델은 아래와 같은 단계를 통해 모델링을 진행하게 된다.
	프로젝트 계획 
			-> 업무 분석 
					-> 시스템 설계 
							-> 시스템 구현 
									-> 테스트 
										    -> 유지,보수
순서대로 진행.

# 프로젝트 계획 : 앞으로 진행하는 프로젝트를 언제부터 언제까지 진행하는지 매우 상세하게 계획함.
		  프로젝트 진행하는 인원,맡은 업무 등의 명단.
# 업무 분석 : 그 기업이나 기관에 무슨 업무들이 처리되는지, 업무가 처리될 때 어떤 단계가 진행되는 지,
		어느 직원이 무슨 업무를 하는지 등을 파악하는 단계
# 시스템 설계 : DB 모델링을 진행 함. 어떤 테이블을 만들지, 테이블에 데이터는 무엇을 무슨 자료형으로 무		슨 컬럼에 저장할지 등을 설계.
# 시스템 구현 : 설계한 DB를 실제 MySQL 서버에 구축하는 작업임... 테이블을 만들어 내는 작업.
# 테스트 : 만든 프로그램과 DB가 운영되는데 문제가 없는지 가동해봄.
# 유지,보수 : 만든 프로그램과 DB를 실제로 운영하고 모니터링 함.
		문제가 생기는 지 체크하고, 문제점을 해결하는 단계.


# 패스워드 복잡성 : 알파벳 대문자, 소문자, 숫자, 특수문자 중에 3가지 이상을 조합해야 되는 것을 뜻함.

#MySQL에서 계정 관련된 작업을 외부에서 접속했을 때는 못하게 막게 됨.
	>>> 설정 해제하는 방법
		Workbench SQLeditor가서
		grant all on *.* to 'root'@'%' identified by 'P@ssw0rd' with grant option; 입력
							      root 비번
		위 SQL은 root 계정이 외부에서 접속해서 계정관련된 작업을 할 수 있게 만드는 쿼리임.
	위의 쿼리는 외부에서 실행할 수 없음. MySQL 로컬 접속 후에 쿼리를 실행해야 함.(mysql -u root -p)
		위 쿼리를 실행 후에 적용되기 위해서는 리눅스 명령어 systemctl restart mysqld을 사용.

# 계정을 만들 때 Limit to hosts Matching 설정을 진행해야 함.

# Limit to hosts Matching : 해당 계정이 어느 IP에서 접속할 수 있게 할 지를 제한하는 것임.
	%라고 설정하면 제한하지 않겠다는 것. 제한하지 않으면 그 계정을 어느 IP를 가지고도 접속할 수 있		음..(어느 곳에서나 접속 가능)
