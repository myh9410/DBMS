# CURDATE()는 운영체제의 날짜 정보를 알아내줌...

#근데 운영체제의 날짜가 이상하다면 데이터에 날짜를 삽입할 때 잘못된 날짜가 저장 됨..

#MySQL Server에 날짜를 올바르게 설정 해 놓아야 CURDATE가 정확한 날짜를 뽑아준다.

#리눅스 - date 명령어	>>> 운영체제에 설정된 날짜와 시간을 알려줌.


	date +%F	>>> 날짜만 출력
	date +%r	>>> 시간만 출력

# date + "날짜 : %F	시간 : %r"으로 출력할 형태를 지정할 수 있음.

# date -s 를 통해서 운영체제 날짜와 시간을 변경할 수 있음.

#Network Time Protocol (NTP) Service : 현재의 날짜와 시간이 어떻게 되는지 다른 장비를 통해 알아와서 내 운영체제 날짜 시간으로 설정할 수 있는 서비스

# rdate 명령어를 통해 NTP서버와 날짜/시간 정보를 동기화할 수 있음. rdate 명령어는 CentOS최소 설치를 하면 같이 설치가 안됨... yum으로 직접 설치해야 함.

# /etc/resolv.conf 파일은 DNS서버를 지정하는 파일.	>>> nameserver 168.126.63.1
# DNS 서버라는 것이 도메인 주소를 사용할 수 있게 해주는 서버

#nameserver 168.126.63.1	>> 다음의 내용을 추가함.	>>> KT회사의 DNS서버를 이용하겠다고 설정하는 것.

# rdate -s time.bora.net
위의 리눅스 명령어를 통해 지정한 주소와 NTP 서비스를 통해 날짜/시간을 동기화하겠다.

# 모델링

# 모델링이란? 현실세계에 있는 "사물"이나 "작업"을 DBMS의 테이블로 만들어 내는 작업을 뜻함.

#모델링을 통해서 판매하는 상품은 [물품 테이블], 고객들 정보는 [고객 테이블] 등을 만들 수 있음.

# L자형 테이블은 문제점으로 지적되는 형태인데, 데이터 저장 공간이 낭비된다는 문제점이 있음.

# 모델링의 결과는 진행하는 사람마다 결과가 달라질 수 있음.

# 좋은 모델링과 안좋은 모델링

# 외래키(FK)(Foreign Key) - 부모, 자식 관계를 맺기 위해 설정하는 것.
외래키 설정을 통해 2개의 테이블을 서로 연결시켜 놓을 수가 있음.	>> 관계설정을 한다. 라고 함
테이블의 관계를 맺기 위해서는 두 테이블에 동일한 데이터가 존재해야 함.
회원 테이블에 "고객이름", 구매테이블에 "고객이름" 이렇게 두 테이블에 동일하게 존재해야 관계설정 가능.

#테이블은 부모테이블, 자식테이블로 구분되어질 수 있다.
