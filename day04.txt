ls 명령어의 옵션
ls 눌렀을 때 파란색으로 출력 ---> 디렉토리 흰색 ----> 파일
1) -l ---> 자세한 정보를 출력함.
	-l로 봤을 때
	-rw-------
	rw앞에 오는 것
	- 	---> 파일임을 의미
	d	---> 디렉토리임을 의미


2) -a : 지정한 경로의 모든것을 보여줌. 
	파일이나 디렉터리를 숨김처리할 수 있는데 숨김처리 된 것까지 전부 보여줌.
		숨김처리의 이유 - 잘못 건들여서 운영체제에 문제가 발생할 수 있기 때문에   				  안보이게 숨김처리함
	ls -a를 통해 봤을 때 앞에 . 이 찍혀 있으면 숨김처리 된 것이라는 뜻.

	모든 옵션은 섞어서 사용 할 수 있다.
	ex) ls -la ---> 숨김처리된 것들 까지 자세한 정보를 출력해줌.
	ex)자세한 정보 출력의 예시
	root root 135 		 Jul 14 05:20
		  용량(단위 byte)
	ls -l통해 출력되는 용량의 단위는 byte이다.

3)ls-lh ---> h는 l과 항상 같이 써야함.
	h -->용량의 크기에 대한 단위를 byte가 아닌 다른 단위를 이용해서 보여줌.
	(알파벳이 붙지 않은 것은 Byte이고 그 외의 단위는 알파벳을 이용해서 표기해준다.)
	(용량이 정확하지 않을 수 있음)

4) -S ---> 용량이 큰 것부터 순서대로 출력

5) -R ---> 지정한 디렉토리에 있는 것과 함께 하위에 있는 모든 것을 보여달라는 옵션


디렉토리 : 무엇인가를 담을 수 있는 바구니
	윈도우 운영체제에서는 폴더라고 하고, 리눅스 운영체제에서는 디렉토리라고 한다.
디렉토리 사용 목적
	정리 정돈
1)mkdir /경로/만들 디렉터리의 이름
	다음 경로에 디렉터리를 만듬
2)mkdir ./b --- 내 현재 위치에 b라는 디렉터리를 만든다.
3)mkdir c c라는 디렉터리를 만든다.


동일한 경로에 여러 디렉토리 만드는 법
mkdir /경로/{만들 디렉토리 이름들}
{} : 중괄호를 사용하는 이유는 동일한 경로에 여러개를 지정하기 위해서임...


mkdir /aaa/bbb/{ccc,ddd/eee}   == /aaa/bbb/ccc와 /aaa/bbb/ddd/eee를 만들어 준 것.

mkdir aaa/bbb/ccc
--->>aaa안에bbb안에ccc를 만들라는 명령어인데 이를 이용하기 위해서는 aaa와 bbb가 이미 만들어져 있어야 한다
mkdir aaa aaa/bbb aaa/bbb/ccc ---> aaa를 만들고 그 다음에 aaa/bbb를 만들고 aaa/bbb/ccc를 만듬 ---> 앞에서부터 순서대로 만든다.
mkdir aaa aaa/bbb aaa/bbb/ccc == mkdir {aaa,aaa/bbb,aaa/bbb/ccc} == mkdir -p aaa/bbb/ccc

-p : aaa aaa/bbb aaa/bbb/ccc 와 같이 불편하게 적는 것을 피하기 위한 명령어
mkdir -p aaa/bbb/ccc --->ccc를 만드는데 aaa와 bbb가 존재하지 않으면 자동으로 만들어주면서 ccc를 만듬


rmdir a --->a라는 디렉토리를 지움.
rmdir 명령어는 비어있는 디렉토리를 지우는 명령어이다.



파일 : 내용을 가지고 있는 데이터
	touch ---> 파일을 만드는 명령어
	사용 방법은 mkdir이나 rmdir이랑 같음.
